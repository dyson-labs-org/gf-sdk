# Create invoice exampleThis folder contains `example.ts`, the canonical invoice-creation example for gf-sdk.`baseUrl` is the portal origin (customer-facing), for example `https://btcpay.dyson-labs.com`.`apiPrefix` is the portal API prefix; production uses `/portal`.The SDK always talks to the portal API root, never `/webhook`.The portal runs on a remote VPS only (170.75.173.239). Local portal execution is not supported.It is NOT a BTCPay webhook URL (the `/webhook` endpoint is internal-only).`/portal-staging` is not currently exposed; staging is TBD unless explicitly provided.BTCPay webhooks and Greenfield APIs are internal-only.No admin token is required.No BTCPay credentials are required.Invoice creation uses `POST {baseUrl}{apiPrefix}/session/start` (production: `/portal/session/start`) and does not require `PORTAL_ADMIN_TOKEN`; only `GF_PORTAL_BASE_URL` and `GF_PORTAL_API_PREFIX`.## Run```bashpnpm installpnpm -C examples/create-invoice start```You can also pass a base URL explicitly:```bashGF_PORTAL_BASE_URL=https://btcpay.dyson-labs.com GF_PORTAL_API_PREFIX=/portal pnpm -C examples/create-invoice startpnpm -C examples/create-invoice start -- --baseUrl=https://btcpay.dyson-labs.com --apiPrefix=/portal```## Switch to productionProduction is the default. If a staging base URL is provided later, update `example.ts`:```tsconst DEFAULT_BASE_URL = "https://btcpay.dyson-labs.com"; // PRODUCTION (default)// const DEFAULT_BASE_URL = "<staging base URL>"; // STAGING (TBD)const DEFAULT_API_PREFIX = "/portal";```
